Create a session checkpoint at .agent/checkpoint.md.

A checkpoint captures the current state of the session so it can be resumed cleanly after a context reset, restart, or mode switch.

Gather and write the following to .agent/checkpoint.md:

1. **Goal**: Read .agent/goal.md for the active goal. If no goal file exists, use the best summary of what we are working toward.
2. **Current Mode**: The currently active agent mode (plan/build/compose/default).
3. **Active Tasks**: List all in_progress and todo tasks from the task graph.
4. **Files Inspected**: List up to 30 files that have been read or analyzed in this session.
5. **Files Modified**: List all files that have been written or edited in this session.
6. **Commands Run**: List the last 20 shell commands run with their exit codes and a one-line summary of what each did.
7. **Key Decisions**: List important architectural, implementation, or design decisions made in this session.
8. **Open Questions**: List any unresolved questions or blockers.
9. **Next Action**: The single most important next step to take when resuming.

$ARGUMENTS

After writing, confirm: "Checkpoint written to .agent/checkpoint.md"
